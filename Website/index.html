"Experience in Azure Cloud Engineering"

In my Azure cloud resume challenges, I encountered various tasks and utilized different Azure services to accomplish them. Firstly, I set up a static website by leveraging the knowledge I gained from similar challenges in Google Cloud Platform (GCP) and Amazon Web Services (AWS) resume challenges. This allowed me to apply my previous experience and adapt it to Azure.

store the website, I utilized a storage account provided by Azure. Additionally, I utilized Front Door CDN to point to the storage account, enabling efficient content delivery and improved website performance.

For tracking visitor count, I employed a function app in Azure. By connecting it to a Cosmos DB, I was able to store and retrieve the necessary data. The function app was triggered by a specific link generated by the function app itself, allowing me to seamlessly connect it to the website.

To streamline the automation and CI/CD (Continuous Integration/Continuous Deployment) processes, I implemented GitHub Actions and Terraform. These tools facilitated the automation of front-end tasks. Currently, I am in the process of automating the backend tasks, utilizing the power of these tools to ensure efficient and reliable deployment of my Azure resources.

Overall, my Azure cloud resume challenges involved setting up a static website, storing it in a storage account and utilizing Front Door CDN for optimal content delivery. I also implemented a visitor count functionality using a function app connected to Cosmos DB. Lastly, I am utilizing GitHub Actions and Terraform to automate the deployment process, both for the front-end and back-end components of the project.
